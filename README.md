# Homelab Dashboard

A self-hosted dashboard for monitoring your server and Docker containers,
with built-in **Login / Sign Up** authentication.

---

## Quick start

```bash
git clone https://github.com/M4riosDev/Homelab-Dashboard.git
cd homelab-dashboard
bash setup.sh
```

The setup script will:

1. Check that Docker is installed
2. Generate a secure random `JWT_SECRET` and save it to `.env`
3. **Ask you to choose a username, name and password** for the first admin account
4. Build and start the containers
5. Wait for the backend to be healthy
6. Create your account automatically via the API

Then open the URL it prints and sign straight in — no Sign Up screen needed.

---

## Manual setup

```bash
# 1. Generate a secret
echo "JWT_SECRET=$(openssl rand -hex 32)" > .env

# 2. Start
docker compose --env-file .env up -d --build

# 3. Sign up at the dashboard
open http://localhost:8080
```

---

## Authentication

- **Sign Up** — name, username, password (min 6 chars)
- **Sign In** — username + password → 24h JWT session
- **Sign Out** — button in the dashboard header
- Passwords are hashed with bcrypt; tokens are signed with HS256
- All API routes require a valid token (401 otherwise)
- User accounts are persisted in `/data/users.json` inside the container

---

## Environment variables

| Variable         | Default                      | Description                             |
|------------------|------------------------------|-----------------------------------------|
| `JWT_SECRET`     | *(auto-generated by setup)*  | Secret for signing JWTs — keep it safe |
| `PORT`           | `3001`                       | Backend port                            |
| `USERS_FILE`     | `/data/users.json`           | Where user accounts are stored          |
| `SHORTCUTS_FILE` | `/data/shortcuts.json`       | Where shortcuts are stored              |
